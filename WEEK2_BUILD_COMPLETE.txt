ARCHITECTURE
================================================================================

Analysis Pipeline Flow:
  
  /api/analyze (POST)
    ├─ Input: recordingId, videoUrl, audioUrl, slideUrls
    ├─ Parallel Execution:
    │  ├─ analyzeVideoWithMediaPipe() ~500ms
    │  ├─ analyzeAudioWithDeepgram() ~800ms
    │  ├─ analyzeSlides() ~600ms
    │  └─ analyzeArgumentStructure() ~700ms
    ├─ Bottleneck: ~850ms total (due to Promise.all)
    └─ Output: feedback + analysisResults
    
  /app/analysis/page.tsx
    ├─ Calls /api/analyze on mount
    ├─ Displays 4 progress bars (one per engine)
    ├─ Animates progress smoothly
    └─ Redirects to /feedback on completion
  
  /app/feedback/page.tsx
    ├─ Loads feedback from sessionStorage
    ├─ Displays overall score (0-100)
    ├─ Shows delivery/argument/slides breakdown
    ├─ Expands sections with FeedbackSection component
    └─ Provides re-record and share options

Score Calculation:
  Delivery = 35% eye contact + 35% movement + 15% expression + 15% pacing
  Argument = Direct from Claude scoring
  Slides = 40% readability + 35% density + 25% consistency
  Overall = (Delivery + Argument + Slides) / 3
